# Investigar lead {#investigate-lead}

Descubra por que alguém não entrou em MQL, se qualificou para um programa ou avançou pelo ciclo de vida.

Para visualizar a lista completa de cenários de investigação, clique na seta para baixo à direita.

CAPTURA DE TELA

| **Title** | Descrição |
|----------|------------|
| **Investigação MQL** | Fornece detalhes sobre por que um lead específico não foi qualificado, parou de se envolver ou ficou preso. |
| **Qualificação do programa** | Fornece detalhes sobre por que um cliente potencial não se qualificou para um programa de aprendizado. |
| **Progressão do ciclo de vida** | Para ser usado para explorar por que um lead está preso em um estágio por mais tempo do que o esperado. |
| **Análise de engajamento** | A ser usado quando um cliente potencial parar de abrir ou clicar em emails. |
| **Jornada da campanha** | Para ser usado para visualizar o caminho completo e os pontos de contato de um lead na Jornada. |
| **Qualidade dos dados** | A ser usado quando houver uma possível expectativa de que dados incorretos estejam causando um problema com o lead. |
| **Transmissão de vendas** | Para ser usado quando as Vendas rejeitaram um cliente potencial e não fizeram o acompanhamento. |
| **Atribuição** | Ser usado para explorar o que influenciou um cliente potencial que levou à geração de uma oportunidade. |
| **Declínio de pontuação** | Para investigar por que uma pontuação de lead caiu inesperadamente. |

>[!TIP]
>
>Para exibir rapidamente uma lista de todos os nove cenários, digite `/lead investigation` na janela de prompt na parte superior.
